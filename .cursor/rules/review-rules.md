Ao revisar análises técnicas externas:

* não gerar nova análise genérica do projeto;
* tratar o documento informado como hipótese inicial;
* validar cada achado contra o projeto real;
* classificar cada ponto como:

  * Confirmado
  * Parcialmente confirmado
  * Não confirmado
  * Possível alucinação
* não implementar código durante a validação;
* priorizar evidência concreta por arquivo, módulo ou componente;
* só sugerir melhoria após validar o problema.
